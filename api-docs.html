1. Pendahuluan & Gambaran Umum

JDI REST API adalah layanan API backend berbasis RESTful yang dikembangkan menggunakan framework Express.js (Node.js) dan didukung oleh database PostgreSQL. API ini dirancang untuk menyajikan data secara cepat dan efisien terkait informasi karyawan (employees), kehadiran harian (daily attendance), kehadiran bulanan (monthly attendance), serta catatan sertifikasi pelatihan karyawan (training).

Seluruh endpoint yang menyajikan data sensitif dilindungi dengan protokol keamanan standar, dilengkapi dengan kompresi gzip/brotli untuk performa transfer data optimal, serta optimasi query database melalui indexing pada kolom sorting utama.

2. Autentikasi & Konfigurasi Dasar

Untuk menjaga keamanan data, semua endpoint API kecuali endpoint kesehatan status (/status) memerlukan autentikasi. Skema autentikasi yang digunakan adalah HTTP Basic Authentication.

Klien harus mengirimkan header HTTP berikut pada setiap request: 

Authorization: Basic <Base64(username:password)>

Konfigurasi Kredensial Default:

Secara default, kredensial autentikasi API dikonfigurasi melalui environment variables (.env):

  • Username: JDI-API (diatur via BASIC_AUTH_USER)
  • Password: JDI-GO2026 (diatur via BASIC_AUTH_PASS)

Contoh Pembuatan Header Autentikasi (Node.js / JS)

const username = 'JDI-API';
const password = 'JDI-GO2026';
const credentials = Buffer.from(`${username}:${password}`).toString('base64');
const headers = {
  'Authorization': `Basic ${credentials}`,
  'Content-Type': 'application/json'
};

5. Indexing Database & Optimalisasi Performa

Untuk menjaga performa query agar tetap responsif meskipun data bertumbuh hingga jutaan baris, telah dibuat database indexes pada kolom-kolom yang digunakan dalam klausul ORDER BY dan WHERE:

Nama Index Database Tabel Target Kolom Target Tujuan Optimalisasi
idx_tb_employee_created_at tb_employee created_at DESC Mempercepat query paginasi default pada daftar karyawan yang diurutkan dari data terbaru.
idx_tb_attendance_daily_shift_start_last_update tb_attendance_daily SHIFT_START DESC, LAST_UPDATE DESC Meningkatkan performa sorting multi-kolom saat sinkronisasi kehadiran harian.
idx_tb_attendance_monthly_shift_start_last_update tb_attendance_monthly SHIFT_START DESC, LAST_UPDATE DESC Meningkatkan performa sorting multi-kolom saat memuat ringkasan bulanan.
idx_tb_training_created_at tb_training CREATED_AT DESC Meningkatkan performa query data pelatihan terbaru.

6. Penanganan Error (Error Handling)

API ini menggunakan kode status HTTP standar untuk menunjukkan keberhasilan atau kegagalan suatu request. Jika terjadi error, sistem akan merespon dengan format JSON standar:

Format Respon Error
{
  "error": "Pesan deskripsi kesalahan detail"
}
HTTP Code Status Error Penyebab & Solusi
401 Unauthorized Header Authorization tidak disertakan, format Basic Auth salah, atau kombinasi username/password salah. Periksa kredensial integrasi Anda.
404 Not Found Endpoint/route URL yang diminta tidak dikenali oleh Express.js. Pastikan path URL sudah benar.
500 Internal Server Error Terjadi kendala pada backend server, seperti koneksi database PostgreSQL yang terputus atau sintaks query yang bermasalah.
